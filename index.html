30-Day Guide: Full-Stack App on AWS with EKS, AngularJS, Node.js, and PostgreSQL

Hosted on GitHub Pages: Follow the daily steps below to learn AWS, AngularJS, Node.js, and microservices from scratch!

Day 1-2: AWS Basics

Learning Goals

  • Understand core AWS services: EC2, S3, IAM, VPC.
  • Launch an EC2 instance and create an S3 bucket.

Step-by-Step Instructions

  1. Create an AWS Account
    - Go to AWS Free Tier and sign up.
    - Verify your email and set up billing alerts to avoid unexpected charges.
  2. Launch an EC2 Instance
    - Navigate to EC2 Dashboard → Launch Instance.
    - Choose Amazon Linux 2 AMI (free tier eligible).
    - Select t2.micro instance type.
    - Configure security groups to allow SSH (port 22) and HTTP (port 80).
    - Download the key pair (.pem file) and save it securely.
    - Connect to the instance via SSH:
    ssh -i your-key.pem ec2-user@your-instance-public-ip
  3. Create an S3 Bucket
    - Go to S3 Dashboard → Create Bucket.
    - Choose a unique name (e.g., my-learning-bucket).
    - Enable Static Website Hosting under the bucket properties.
    - Upload a file (e.g., index.html) and make it publicly accessible.

Day 3-4: Node.js Basics

Learning Goals

  • Learn Node.js fundamentals and build a REST API with Express.js.

Step-by-Step Instructions

  1. Install Node.js
    - Download and install Node.js from nodejs.org.
    - Verify the installation:
    node -v
npm -v
  2. Create a Simple HTTP Server
    - Create a file named server.js:
    const http = require('http');
const server = http.createServer((req, res) => {
  res.end('Hello, World!');
});
server.listen(3000, () => console.log('Server running on port 3000'));
    - Run the server:
    node server.js
    - Open your browser and navigate to http://localhost:3000.
  3. Build a REST API with Express.js
    - Install Express:
    npm install express
    - Create a file named app.js:
    const express = require('express');
const app = express();
app.get('/', (req, res) => res.send('Hello, World!'));
app.listen(3000, () => console.log('Server running on port 3000'));
    - Run the app:
    node app.js

Day 5-6: AngularJS Basics

Learning Goals

  • Learn AngularJS fundamentals: directives, controllers, and data binding.

Step-by-Step Instructions

  1. Create a Simple AngularJS App
    - Create an index.html file:
    <!DOCTYPE html>
<html ng-app="myApp">
<head>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.2/angular.min.js"></script>
</head>
<body ng-controller="myCtrl">
  <input ng-model="name" placeholder="Enter your name">
  <h1>Hello, {{name}}!</h1>
  <script>
    angular.module('myApp', []).controller('myCtrl', function($scope) {
      $scope.name = 'World';
    });
  </script>
</body>
</html>
    - Open the file in your browser to see the app in action.

Day 7: Microservices Concepts

Learning Goals

  • Understand microservices architecture and its benefits.

Step-by-Step Instructions

  1. What are Microservices?
    - Microservices are small, independent services that work together to form a larger application.
    - Each service handles a specific functionality (e.g., user authentication, payment processing).
  2. Benefits of Microservices
    - Scalability: Scale individual services as needed.
    - Flexibility: Use different technologies for different services.
    - Fault Isolation: A failure in one service doesn’t affect others.

Day 8-9: AWS EKS Setup

Learning Goals

  • Set up an EKS cluster and deploy a Node.js app.

Step-by-Step Instructions

  1. Install AWS CLI and eksctl
    - Install AWS CLI: AWS CLI Installation.
    - Install eksctl: eksctl Installation.
  2. Create an EKS Cluster
    - Run the following command:
    eksctl create cluster --name my-cluster --region us-east-1
    - This will take 10-15 minutes to complete.
  3. Deploy a Node.js App
    - Create a Kubernetes deployment file named deployment.yaml:
    apiVersion: apps/v1
kind: Deployment
metadata:
  name: nodejs-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nodejs-app
  template:
    metadata:
      labels:
        app: nodejs-app
    spec:
      containers:
      - name: nodejs-app
        image: your-docker-image:latest
        ports:
        - containerPort: 3000
    - Apply the deployment:
    kubectl apply -f deployment.yaml

Day 10-11: PostgreSQL on AWS

Learning Goals

  • Set up PostgreSQL on AWS RDS and connect it to a Node.js app.

Step-by-Step Instructions

  1. Create an RDS Instance
    - Go to RDS Dashboard → Create Database.
    - Choose PostgreSQL and configure the instance.
    - Set the master username and password.
  2. Connect to PostgreSQL
    - Use a PostgreSQL client (e.g., psql) to connect to the database.
    - Create a table:
    CREATE TABLE users (id SERIAL PRIMARY KEY, name VARCHAR(50));
