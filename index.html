Day 1-2: AWS Basics

What it means

AWS (Amazon Web Services) is a cloud computing platform that provides services like virtual servers (EC2), storage (S3), and databases (RDS). Today, you'll learn the basics of AWS and set up your first EC2 instance and S3 bucket.

What you should learn

  • What is AWS and why is it used?
  • How to launch an EC2 instance (virtual server).
  • How to create and configure an S3 bucket (storage).

Step-by-Step Instructions

  1. Create an AWS Account
    - Go to AWS Free Tier and sign up.
    - Verify your email and set up billing alerts to avoid unexpected charges.
  2. Launch an EC2 Instance
    - Navigate to EC2 Dashboard → Launch Instance.
    - Choose Amazon Linux 2 AMI (free tier eligible).
    - Select t2.micro instance type.
    - Configure security groups to allow SSH (port 22) and HTTP (port 80).
    - Download the key pair (.pem file) and save it securely.
    - Connect to the instance via SSH:
    ssh -i your-key.pem ec2-user@your-instance-public-ip
  3. Create an S3 Bucket
    - Go to S3 Dashboard → Create Bucket.
    - Choose a unique name (e.g., my-learning-bucket).
    - Enable Static Website Hosting under the bucket properties.
    - Upload a file (e.g., index.html) and make it publicly accessible.

Lab Instructions

  1. Launch an EC2 instance and connect to it using SSH.
  2. Create an S3 bucket and upload a file.
Day 3-4: Node.js Basics

What it means

Node.js is a runtime environment that allows you to run JavaScript on the server. It’s used to build scalable and fast backend applications. Today, you'll learn the basics of Node.js and build a simple REST API.

What you should learn

  • What is Node.js and how does it work?
  • How to create a simple HTTP server.
  • How to build a REST API using Express.js.

Step-by-Step Instructions

  1. Install Node.js
    - Download and install Node.js from nodejs.org.
    - Verify the installation:
    node -v
npm -v
  2. Create a Simple HTTP Server
    - Create a file named server.js:
    const http = require('http');
const server = http.createServer((req, res) => {
  res.end('Hello, World!');
});
server.listen(3000, () => console.log('Server running on port 3000'));
    - Run the server:
    node server.js
    - Open your browser and navigate to http://localhost:3000.
  3. Build a REST API with Express.js
    - Install Express:
    npm install express
    - Create a file named app.js:
    const express = require('express');
const app = express();
app.get('/', (req, res) => res.send('Hello, World!'));
app.listen(3000, () => console.log('Server running on port 3000'));
    - Run the app:
    node app.js

Lab Instructions

  1. Create a simple HTTP server using Node.js.
  2. Build a REST API using Express.js.
Day 5-6: AngularJS Basics

What it means

AngularJS is a JavaScript framework for building dynamic web applications. It allows you to create single-page applications (SPAs) with features like two-way data binding and dependency injection.

What you should learn

  • What is AngularJS and how does it work?
  • How to create a simple AngularJS app.
  • How to use directives, controllers, and data binding.

Step-by-Step Instructions

  1. Create a Simple AngularJS App
    - Create an index.html file:
    <!DOCTYPE html>
<html ng-app="myApp">
<head>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.2/angular.min.js"></script>
</head>
<body ng-controller="myCtrl">
  <input ng-model="name" placeholder="Enter your name">
  <h1>Hello, {{name}}!</h1>
  <script>
    angular.module('myApp', []).controller('myCtrl', function($scope) {
      $scope.name = 'World';
    });
  </script>
</body>
</html>
    - Open the file in your browser to see the app in action.

Lab Instructions

  1. Create a simple AngularJS app with two-way data binding.
Day 7: Microservices Concepts

What it means

Microservices is an architectural style where an application is built as a collection of small, independent services. Each service handles a specific functionality and communicates with others via APIs.

What you should learn

  • What are microservices and why are they used?
  • How microservices differ from monolithic architecture.
  • Key benefits of microservices.

Step-by-Step Instructions

  1. What are Microservices?
    - Microservices are small, independent services that work together to form a larger application.
    - Each service handles a specific functionality (e.g., user authentication, payment processing).
  2. Benefits of Microservices
    - Scalability: Scale individual services as needed.
    - Flexibility: Use different technologies for different services.
    - Fault Isolation: A failure in one service doesn’t affect others.

Lab Instructions

  1. Research and write down the differences between monolithic and microservices architecture.
Day 8-9: AWS EKS Setup

What it means

AWS EKS (Elastic Kubernetes Service) is a managed Kubernetes service that allows you to run containerized applications. Kubernetes is a tool for automating deployment, scaling, and management of applications.

What you should learn

  • What is Kubernetes and why is it used?
  • How to set up an EKS cluster on AWS.
  • How to deploy a Node.js app on EKS.

Step-by-Step Instructions

  1. Install AWS CLI and eksctl
    - Install AWS CLI: AWS CLI Installation.
    - Install eksctl: eksctl Installation.
  2. Create an EKS Cluster
    - Run the following command:
    eksctl create cluster --name my-cluster --region us-east-1
    - This will take 10-15 minutes to complete.
  3. Deploy a Node.js App
    - Create a Kubernetes deployment file named deployment.yaml:
    apiVersion: apps/v1
kind: Deployment
metadata:
  name: nodejs-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nodejs-app
  template:
    metadata:
      labels:
        app: nodejs-app
    spec:
      containers:
      - name: nodejs-app
        image: your-docker-image:latest
        ports:
        - containerPort: 3000
    - Apply the deployment:
    kubectl apply -f deployment.yaml

Lab Instructions

  1. Set up an EKS cluster and deploy a Node.js app.
